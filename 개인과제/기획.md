## 1. 삼일 신입·인턴이 실제로 하는 일 (감사 스태프 기준)

Big 4 감사 신입(A1)과 인턴에게 떨어지는 일의 카탈로그. 세무·딜 배치면 일부 달라지지만 문서·대사 중심이라는 본질은 같다.

| # | 업무 | 내용 | 반복성 |
|---|---|---|---|
| 1 | 증빙 대사(바우칭) | 전표·원장 항목 ↔ 세금계산서·계약서·입금증 대조, 금액·날짜·거래처 확인 | 매우 높음 |
| 2 | 조서 롤포워드 | 전기 감사조서를 당기 폴더로 복사하고 날짜·숫자·참조 갱신 | 높음 |
| 3 | 잔액조회서(컨펌) 관리 | 은행·거래처 조회서 리스트업, 발송, 회수 체크, 미회수 팔로업 | 높음 |
| 4 | TB 매핑·리드시트 | 시산표를 조서 체계로 매핑, 합계 검산(footing) | 높음 |
| 5 | FS tie-out | 재무제표·주석 숫자가 조서와 일치하는지 대조, 오탈자 확인 | 시즌 집중 |
| 6 | PBC 자료 정리 | 고객사가 보낸 파일 분류·리네이밍, 누락 자료 체크리스트 | 높음 |

| 8 | 영문 번역 | 글로벌 보고용 문서·이메일 번역 | 중간 |
| 9 | 회의록·주간보고·타임시트 | 미팅 노트, 주간 업무 보고, 차지 코드 입력 | 높음 |
| 10 | 교육·이러닝 정리 | 교육 내용 노트, 용어 학습 | 초기 집중 |

## 3. 1픽: 바우칭 에이전트 (증빙-장부 자동 대사)

신입 야근의 최대 지분인 단순 대조 노동을 자동화한다.

### 기능

1. 증빙 이미지·PDF(세금계산서·영수증·계약서) 업로드 → 비전 LLM이 거래처·금액·일자·품목 추출
2. 원장 엑셀과 자동 매칭: 완전일치 / 유사(금액 반올림·날짜 어긋남) / 불일치 3분류
3. 불일치 건은 사유 추정 코멘트 생성("부가세 포함가 차이로 추정" 등), 신입은 예외 건만 확인
4. 결과를 조서 형식 엑셀로 출력 + 합계 검산(footing)
5. (확장) tie-out 모드: 재무제표 PDF 숫자 ↔ 조서 대조

### 데모 시나리오 (민감정보 제로)

가짜 영수증·세금계산서 10장 + 가짜 원장 엑셀 생성 → 업로드 → 8건 자동 매칭, 2건 예외 플래그+사유 코멘트 → 조서 출력. 3분 영상으로 완결.

### 왜 이게 이기는 픽인가

- 과제 문구("신입으로서 삼일 생활에 도움이 될") 정중앙: 감사 신입의 밤샘 노동 그 자체
- 가산점 기준("업무 자동화를 효과적으로 구현")에 정면 대응, 기대효과 정량화 쉬움(건당 3〜5분 → 수 초)
- 삼일 전략과 정렬: Document AI·AI Accountant의 방향과 같은 결이되, "신입 개인용 경량 도구"라 중복이 아니라 보완
- 판단이 필요한 업무라 비전+LLM이 필수(단순 OCR·매크로로 안 되는 이유를 설명 가능 → AI 적합성 방어)

### 구현 계획

- Claude API(비전 포함) + 간단한 웹 UI(Streamlit). 이 저장소 개인과제/ 폴더에서 개발
- 데모용 가짜 데이터 생성 스크립트 포함
- 착수는 조별 사전조사 마감(7/15) 이후, 데모 촬영 7/25 전후

### 리스크·확인 사항

- 하계인턴이 감사팀 실무에 실제 배치되는지는 미확인. 배치가 교육 중심이면 소재의 실감을 현직 선배(버디·멘토) 한마디로 검증해 발표에 인용할 것
- 실데이터 사용 금지: 데모는 전량 합성 데이터로만
