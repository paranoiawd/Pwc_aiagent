# 개인과제 기획: 무엇을 만들 것인가 (딜 부문 기준)

상태: 아래 1픽은 리서치 기반 가설이다. 버디·멘토에게 딜 신입 시절 병목을 확인한 뒤 픽을 확정하고 구현에 들어간다. 뇌피셜로 만들지 않는다.

## 0. 확정 프로세스와 버디 질문

순서: 버디 커피챗(10분, 〜7/17 목표) → 픽 확정 → 구현(1주) → 데모 촬영(7/25 전후).

버디에게 물어볼 것:

- 신입 1년차 때 시간을 제일 많이 잡아먹은 업무가 뭐였어요?
- 딜 신입의 단순 반복 업무 중 "이건 진짜 자동화됐으면" 싶은 것 하나만 꼽으면요?
- (객관식) 데이터룸(VDR) 자료 정리·인덱싱 / GL·TB 데이터 클렌징 / 데이터북 업데이트 / Q&A 리스트 관리 / 산업 리서치·벤치마킹 / 보고서 tie-out / 번역 중 제일 지옥은요?
- 그 업무, 요즘 사내 툴이 해결해주는 부분이 있어요? (기존 툴과 중복 회피 확인, 중요)
- 신입이 그런 도구를 만들어오면 뭐가 제일 쓸모 있을 것 같아요?

버디 답이 아래 1픽과 다르면 그 답을 따르고, 카탈로그·스코어링 틀과 구현 계획(합성 데이터 + Claude API + 경량 UI)은 그대로 재사용한다.

## 1. 딜 부문 신입·인턴이 실제로 하는 일 (FDD·Valuation 보조 기준)

| # | 업무 | 내용 | 반복성 |
|---|---|---|---|
| 1 | 데이터룸(VDR) 자료 정리 | 수백 개 문서 다운로드, 폴더 분류, 파일명 정리, 인덱스 작성, 신규 업로드 추적 | 매우 높음 |
| 2 | GL·TB 클렌징 | 대상회사 원장·시산표 덤프를 분석용으로 가공, 계정 매핑, 월별 정합성 맞추기 | 매우 높음 |
| 3 | 데이터북 업데이트 | 월별 손익·BS 추이표, 매출 분해(제품·거래처별), 엑셀 표 생산 | 높음 |
| 4 | QoE 보조 | 비경상·일회성 항목 후보 찾기(일회성 매출, 특수관계자 거래 등) | 높음 |
| 5 | Q&A 트래커 관리 | 매도자 측 질의 리스트 취합·발송·답변 정리, 버전 관리 | 높음 |
| 6 | 산업·시장 리서치 | 시장 규모·경쟁사·뉴스 조사, 보고서 산업 섹션 초안 | 높음 |
| 7 | 벤치마킹·멀티플 수집 | DART 공시에서 유사기업 재무 수집, 배수 계산 | 중간 |
| 8 | 보고서 tie-out | FDD 리포트 숫자 ↔ 데이터북 대조, 오탈자·포맷 확인 | 시즌 집중 |

| 10 | 회의록·주간보고 | 킥오프·경영진 인터뷰 노트, 팀 보고 | 높음 |

## 3. 1픽(가설): 실사 데이터룸 에이전트

딜 실사의 상징적 병목인 "수백 개 문서에서 정보 찾기"를 자동화한다.

### 기능

1. 데이터룸 문서 더미(PDF·엑셀·워드) 업로드 → 자동 분류(재무·세무·법무·인사·영업), 파일명 표준화, 인덱스 시트 생성
2. 자연어 질문("최근 3년 리스 계약 조건은?") → 관련 문서·페이지를 찾아 근거와 함께 답변(RAG)
3. 자료 요청 리스트와 대조해 누락 문서 자동 체크
4. (확장) 신규 업로드 문서 요약 브리핑

### 데모 시나리오 (민감정보 제로)

가상 대상회사의 합성 문서 20〜30개(재무제표·계약서·급여대장 등을 LLM으로 생성) → 업로드 → 자동 인덱스 생성 장면 → 질문 3개에 출처와 함께 답하는 장면 → 누락 체크리스트 출력. 3분 영상.

### 왜 이게 1픽인가 (가설)

- 딜 신입의 실제 최다 노동(VDR 정리·문서 뒤지기)을 정면 타격
- LLM 필수성 방어가 쉬움: 문서 분류·내용 이해·출처 추적은 룰베이스로 불가능
- "딜 현장의 축소판"이라 심사자(삼일)가 즉시 알아봄. 가산점 기준(업무 자동화 효과적 구현)에 직결
- 합성 데이터로 완결되어 보안 이슈 없음

### 구현 계획

- Claude API + 경량 웹 UI(Streamlit). 이 저장소 개인과제/ 폴더에서 개발
- 합성 문서 생성 스크립트 포함
- 착수는 버디 확인 후. 데모 촬영 7/25 전후

### 보안 원칙 (절대 준수)

- 실사 자료는 NDA·미공개중요정보(자본시장법)·VDR 반출 통제 대상. 실제 딜 자료·고객 자료를 외부 AI(개인 Claude·ChatGPT 포함)에 입력하는 것 금지
- 데모는 전량 합성 데이터로만 제작
- 발표 프레임: "데모는 개인 API + 합성 데이터, 실배포는 사내 승인 환경(Chat X 온프레미스·사내 Azure 테넌트)으로 LLM 레이어 교체 가능한 구조" → 보안을 설계 포인트로 제시
- 버디 확인 질문에 추가: 사내에서 허용된 AI 사용 범위가 어디까지인지

### 리스크·확인 사항

- 딜 부문에 이미 VDR 분석 사내 툴이 있는지 버디에게 반드시 확인(있으면 2픽 GL 클렌징으로 전환)
- 인턴 실무 범위가 리서치 중심이면 소재 실감을 버디 발언 인용으로 보강
