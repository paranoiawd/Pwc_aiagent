# 0715 실습 플랜: LG전자 구독서비스 사례 슬라이드 (BMR·XaaS)

## 목표
- A사 미팅용 BMR 설파 자료 중 1장. XaaS 대표 사례로 LG전자 구독서비스를 소개한다
- 슬라이드가 주는 결론: A사도 구독 서비스를 검토해야 한다
- 최종 산출물: 클로드 디자인에 전달할 영어 덱 생성 프롬프트 1건
- 최종 산출물 전까지 모든 작업은 한국어로 진행

## 산출물
- `조사노트.md`: LG전자 구독서비스 사실 관계, 수치, 출처 (한국어)
- `스토리라인.md`: 슬라이드 1장 구성안과 메시지 (한국어)
- `deck_prompt_v0.1.md`: 클로드 디자인용 덱 생성 프롬프트 (영어, 최종)

## 단계
1. **자료 조사**: 웹 조사로 사실 확보 후 `조사노트.md` 작성
   - 개요: 서비스 구조(구독 기간·케어서비스 결합), 대상 품목, 출시 연혁, 사업 편제
   - 성과: 연도별 매출액, 성장률, 가전 매출 내 비중, 구독 가입 비중, 해외 확장
   - 수치마다 출처·기준 시점 기록, 상충 수치는 교차 검증
2. **스토리라인 설계**: `스토리라인.md` 작성
   - 액션 타이틀(사례의 결론 한 문장), 개요 블록, 성과 블록(차트 스펙), A사 시사점 블록
   - 설득 흐름: 소유에서 사용으로의 시장 변화, LG의 전환, 성과 입증, A사 시사점
3. **콘텐츠 확정**: 수치 최종 검증, 메시지 정제, `한국어_문체_지침.md` 교정 절차 적용
4. **영어 프롬프트 작성**: `deck_prompt_v0.1.md`
   - 포함 요소: 슬라이드 목적·청중, 레이아웃 지시, 확정 텍스트 전문, 차트 데이터·형식, 톤앤매너(전략 컨설팅 스타일)
   - 프롬프트 단독으로 디자인 작업이 가능하도록 자기완결적으로 작성
   - 수치는 전부 구체적 값과 출처를 프롬프트 본문에 직접 기입. 외부 파일·저장소 참조 지시 금지(클로드 디자인의 외부 참조가 불안정)
5. **검수**: 수치 정합성, 영어 표현, 프롬프트 실행 가능성 점검 후 출고

## 슬라이드 1장 구성안 (초안, 조사 후 확정)
- 상단: 액션 타이틀
- 좌측: 개요(비즈니스 모델 전환 내용, 서비스 구조, 차별 요소)
- 우측: 성과(연도별 매출 차트, 핵심 지표 2〜3개)
- 하단: A사 시사점 한 줄

## 운영
- 단계마다 커밋(한국어 한 줄 메시지)
- 프롬프트 수정 시 덮어쓰지 않고 새 버전 파일 생성(v0.1 → v0.2)
