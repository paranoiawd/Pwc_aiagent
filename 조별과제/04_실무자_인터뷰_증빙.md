## 2. 소스 A: 조선사 경영기획(원가·실적 관리) 실무자

질문: "조선사 경영기획, 배 한 척의 원가 관리가 궁금합니다" (업무 리듬·자료 취합 / 변경의 반영 과정 / 도구·역량)

답변 요지
- 월초 약 1주일간 월결산·실적 보고에 집중. 전사 실적 취합과 차이 분석에 최다 시간 투입
- ERP로 산출하되 경영진 보고·부서 간 이견 조정은 엑셀 수작업 가공. 설계·구매·생산 제출 데이터의 정합성 검증이 고됨
- 설계 변경·자재가 변동·공정 지연은 일상. 설계 물량 변동 → 구매 단가 변동 → 생산 공수(시수) 변동 순으로 데이터 발생
- 이를 취합해 총예정원가(EAC)를 매월 재산정, 실행예산에 반영
- 가장 힘든 단계는 부서 간 예산 줄다리기(통제하려는 기획 vs 여유를 확보하려는 현장)의 조율
- 자동화는 단순 추출·양식 채우기 수준. 원가 변동의 원인과 향후 리스크를 경영진 언어로 재생산하는 일은 전적으로 사람 몫

직접 인용
1. "총예정원가(EAC)를 매월 재산정하고 실행예산에 반영합니다"
2. "부서 간 이견 조정은 결국 엑셀로 데이터를 내려받아 수작업으로 가공하고 분석하는 경우가 많습니다"
3. "가장 힘든 단계는 부서 간의 예산 줄다리기입니다"
4. "왜 원가가 변동되었으며 향후 리스크는 무엇인지 경영진의 언어로 재생산하는 일은 전적으로 사람의 몫입니다"

## 3. 소스 B: 조선소 생산관리 실무자

질문: "조선소 생산관리, 계획대로 안 될 때 현장은 어떻게 돌아가나요?" (실적 집계 방식 / 지연 인지·대응 / 스마트야드 체감)

답변 요지
- 공정 실적은 MES·ERP로 일부 자동 집계되나 완전 자동화 아님. 현장 완료 입력, 협력업체 보고의 담당자 확인·시스템 반영 등 사람의 확인 과정 필수
- 지연 인지는 시스템 실적·현장 점검 외에 작업반장·협력업체의 선(先)보고 경로 혼재
- 지연 시 원인 분석(자재·인력·설계변경·장비) 후 부서 협의로 만회 계획 수립(추가 인력·순서 조정·잔업)
- 스마트야드·AI 확대 중이나 엑셀 공정 정리, 회의자료 작성, 전화·메신저 확인 업무 여전. 변수 대응은 시스템보다 사람의 판단·조율

직접 인용
1. "완전히 자동화된 것은 아닙니다 … 사람의 확인 과정이 반드시 들어갑니다"
2. "현장 작업반장이나 협력업체에서 먼저 보고하는 경우도 있습니다"
3. "엑셀로 공정을 정리하거나 회의 자료를 만들고 협력업체와 전화나 메신저로 작업 상황을 확인하는 일도 여전히 많습니다"

## 4. 답변 → Pain Point 조립 (과제 템플릿 형식)

관찰사실 체인 (앞단 → 뒷단)
1. 공정 실적 집계에 사람의 확인·수기 입력이 필수 (소스 B)
2. 지연·변경 신호가 작업반장 보고, 전화·메신저 등 비정형 경로로도 전파 (소스 B)
3. 설계 물량 → 구매 단가 → 생산 공수 순으로 변동 데이터가 시차를 두고 발생 (소스 A)
4. 경영기획이 월결산 기간(월 1회, 약 1주)에 엑셀 수작업으로 취합·정합성 검증, EAC를 매월 재산정 (소스 A)
5. 실행예산 반영은 부서 간 협상(예산 줄다리기)으로 조율되고, 원가 변동·리스크의 해석은 전적으로 사람 몫 (소스 A)

Pain Point: 변경 신호의 비정형·순차 전파와 월 단위 수작업 EAC 재산정으로 인한 회계 반영 시차 + 추정 조율 과정의 조직 역학 개입 + 결산 기간 인력 과부하

## 5. 변론 문서(01) 주장 매핑

| 발화 | 지지하는 주장 |
|---|---|
| "EAC를 매월 재산정" (A) | C1 실무 확인. 갱신 주기가 월 배치임을 확정, 상시 검증 부재(C3·C6 전제) |
| 엑셀 수작업 취합·정합성 검증 (A) | C6 (사후·수작업의 규모 한계) |
| 물량 → 단가 → 공수 순차 발생 (A) | C5 (신호의 순차 전파와 시차) |
| 부서 간 예산 줄다리기 (A) | 추정치가 부서 간 협상의 대상이라는 증언. C1·C2 메커니즘의 온건한 실무 방증 |
| 리스크의 경영진 언어 재생산은 사람 몫 (A) | 제안 기능 2(회계 영향 환산)의 현재 공백 |
| 실적 집계에 사람 확인 필수 (B) | C1 입력단·C6 |
| 엑셀·전화·메신저 여전 (B) | E05-5(스마트야드) 반박 보강: 시스템 존재와 현장 실태의 간극 |

주의: 소스 A의 "예산 줄다리기" 발화는 정상적 예산 조율 과정의 설명이다. 분식·왜곡의 증언으로 인용하지 않으며, 추정에 조직 역학이 개입하는 구조의 방증으로만 쓴다.

## 부록: 답변 원문 전문

### 소스 A 원문

한 달 업무 리듬은 매월 초 약 1주일간 진행되는 월결산 및 실적 보고에 집중됩니다. 이때 전사 부서의 실적 데이터를 취합하고 차이(분석)를 규명하는 데 많은 시간을 씁니다. 데이터는 기본적으로 사내 ERP 시스템을 통해 산출되지만, 경영진 보고를 위한 인사이트 도출이나 부서 간 이견 조정은 결국 엑셀로 데이터를 내려받아 수작업으로 가공하고 분석하는 경우가 많습니다. 특히 설계, 구매, 생산 각 부서에서 제출한 데이터의 정합성을 검증하는 과정이 꽤나 고됩니다.

조선업 특성상 건조에 수년이 걸리므로 잦은 설계 변경과 원자재가 변동, 공정 지연은 일상적입니다. 현장에서 변경 사항이 발생하면 설계 부서의 물량 변동, 구매 부서의 단가 변동, 생산 부서의 투입 공수(시수) 변동 데이터가 차례로 발생합니다. 경영기획 및 관리 부서는 이를 취합해 총예정원가(EAC)를 매월 재산정하고 실행예산에 반영합니다. 이 과정에서 가장 힘든 단계는 부서 간의 예산 줄다리기입니다. 원가를 통제하려는 기획 부서와 예산을 더 확보해 여유를 가지려는 현장 부서 간의 입장을 조율하고 합의점을 끌어내는 소통 과정이 어렵고 또 오래 걸립니다.

시스템과 AI가 고도화되면서 단순 데이터 추출이나 반복적인 양식 채우기 등은 점차 자동화되고 있습니다. 하지만 숫자에 담긴 현장의 맥락을 파악하고, 왜 원가가 변동되었으며 향후 리스크는 무엇인지 경영진의 언어로 재생산하는 일은 전적으로 사람의 몫입니다. 따라서 신입사원에게 가장 필요한 역량은 현장 엔지니어들의 고충을 이해하면서도 회사의 목표 방향으로 설득할 수 있는 커뮤니케이션 능력입니다. 더불어 방대한 데이터를 자유자재로 다루는 엑셀 활용 능력은 질문자님의 퇴근시간을 앞당겨줄것이므로 미리미리 연습해두시는 것을 추천합니다.

### 소스 B 원문

조선소 생산관리는 사무실에서 일정만 관리하는 직무가 아니라 현장과 사무실을 계속 오가며 공정이 계획대로 진행되는지를 관리하고 문제를 해결하는 역할이라고 생각하면 됩니다. 하루 업무는 보통 아침 작업회의로 시작해 전날 실적과 당일 작업 계획을 확인하고 이후 현장에 나가 작업 진행 상황을 직접 점검합니다. 작업이 계획대로 진행되는지, 자재는 제때 들어왔는지, 크레인이나 장비 사용에 문제가 없는지, 협력업체 인력이 충분한지 등을 확인하고 문제가 발생하면 관련 부서와 협의해 해결합니다. 공정 실적은 MES나 ERP 같은 생산관리 시스템을 사용하는 회사가 많아 일부는 자동으로 집계되지만 완전히 자동화된 것은 아닙니다. 실제로는 작업 완료 입력을 현장에서 하거나 협력업체가 보고한 내용을 생산관리 담당자가 확인하고 시스템에 반영하는 경우도 많아 사람의 확인 과정이 반드시 들어갑니다. 계획보다 일정이 밀리면 시스템상의 실적과 현장 점검을 통해 바로 확인되는 경우도 있고 현장 작업반장이나 협력업체에서 먼저 보고하는 경우도 있습니다. 지연이 발생하면 왜 지연됐는지 원인을 분석하고 자재 문제인지 인력 부족인지 설계 변경인지 장비 문제인지를 파악한 뒤 생산기술, 구매, 설계, 품질 등 관련 부서와 협의해 만회 계획을 수립합니다. 필요하면 추가 인력을 투입하거나 작업 순서를 조정하고 잔업이나 특근을 통해 일정을 맞추기도 합니다. 스마트야드와 AI는 실제로 점점 확대되고 있지만 아직 모든 업무가 자동화된 것은 아닙니다. 생산 현황 모니터링과 자재 위치 추적, 일부 일정 관리는 시스템을 활용하는 경우가 많지만 엑셀로 공정을 정리하거나 회의 자료를 만들고 협력업체와 전화나 메신저로 작업 상황을 확인하는 일도 여전히 많습니다. 현장은 예상치 못한 변수가 계속 발생하기 때문에 시스템보다 사람의 판단과 조율이 중요한 경우가 많으며, 생산관리는 데이터를 분석하는 능력뿐 아니라 현장에서 여러 부서와 소통하며 문제를 해결하는 역량이 가장 중요한 직무라고 보시면 됩니다.
