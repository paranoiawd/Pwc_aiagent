# 한화오션 실무자 검증 질문 (AI 서비스 확정 가정)

## 목적
익명 인터뷰 3건(소속 미확인)의 pain point가 한화오션에서 사실인지 확정하고, 적대심사가 남긴 공백(공수 실측·입력 문서 실재·보안 제약)을 채운다.

## 우선순위
시간이 짧으면 1·2·3만 묻는다. 경영기획·원가 직군이면 1·3 우선, 생산·현장 직군이면 2·4 우선.

## 질문지

| # | 질문 | 검증 대상 |
|---|---|---|
| 1 | 월결산 때 실적 취합·검증에 몇 명이 며칠 정도 매달리나요? 그중 엑셀 수작업 비중은 어느 정도인가요? | 검증 공수 정량화(기대효과 숫자, 안건 7) |
| 2 | 설계변경이나 자재가 변동 소식은 처음에 어떤 형태로 오나요? 시스템 통지서인가요, 메일·전화·메신저인가요? 정식 문서로 등록되기까지 시간이 걸리는 편인가요? | AI가 읽을 입력 문서의 실재 + 문서화 이전 사각 |
| 3 | 최근 변경이 생겼던 건 하나를 떠올리면, 발생부터 원가 숫자 반영까지 얼마나 걸렸나요? 다음 결산까지 기다렸나요? | 서비스가 줄인다는 시차의 실증, 월 1회 주기 확인 |
| 4 | 지금 시스템(ERP·스마트야드)이 변경을 자동으로 잡아서 알려주기도 하나요? 어디까지 자동이고 어디부터 사람이 하나요? | 기존 사내 투자와의 중복 여부 |
| 5 | 업무 문서를 외부 업체 시스템이나 클라우드에 올리는 건 어느 정도까지 허용되나요? 상선과 함정 쪽이 다른가요? | 온프레미스 필요성, 방산 보안 제약 |
| 6 | (마지막에만) 문서를 상시로 읽고 "이 프로젝트 원가를 다시 봐야 한다"는 알림을 주는 도구가 있다면 도움이 될까요, 오탐이 잦으면 귀찮을까요? 어떤 조건이면 쓰시겠어요? | 컨셉 수용성, 오탐 허용도 |

## 금지
- 회계 민감 용어(분식·미청구공사·충당금·감사)를 먼저 꺼내지 않는다. 상대 입에서 나오면 발화 그대로 인용 확보.
- 가정형 질문은 6번 하나로 제한. 나머지는 과거의 구체 사건만 묻는다.
- 소속·직군·연차를 확인해 기록한다(성명 익명). 04 증빙에 추가.
