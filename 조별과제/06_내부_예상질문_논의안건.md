# 내부 메모: 예상 질문·논의 안건 (공유용 아님)

## 예상 질문과 답

| 질문 | 답 |
|---|---|
| 실적이 사상 최대인데 무슨 문제인가 | 지금 손실을 주장하는 것이 아니다. 주장은 익스포저(왜곡이 쌓였던 계정의 잔액이 2년 만에 두 배, 5조5,061억)다. 대우조선도 3조 원 일괄 인식 직전까지 장부상 흑자로 보였다 |
| 수주 호황에 대금을 인도 때 받는 구조면 미청구공사 느는 게 당연하지 않나 | 맞다. 그래서 부실의 증거로 쓰지 않는다. 주장은 왜곡이 숨을 수 있는 공간의 크기와 그 공간을 상시로 검증하는 장치의 부재다. 보조 신호로 회수 위험 대비 손실충당금이 70억에서 0원이 된 것과 감사보고서의 러시아 신용위험 강조사항을 든다 |
| 결국 회사 숫자로 회사 숫자를 검증하는 것 아닌가 | 아니다. 회사 추정치는 검증 대상이고 판정 기준은 회사 밖 원천(계약 조항·시장 지표·회계기준)이다. 도구는 신호가 추정치에 반영됐는지 대조하고, 미반영 신호의 영향 범위를 계약·시장 근거로 산출한다 |
| 주기적 지정제로 삼일이 다시 감사인이 되면 자기검토 아닌가 | K-SOX 통제의 소유자는 회사라서 도구 제공자와 통제 운영자는 처음부터 분리된다. 재지정 시점에는 운영을 고객 내재화하거나 제3자에 이관하는 계약 구조로 답한다 |
| 잘나가는 회사가 왜 돈을 내고 사나 | 지금 경영진은 한화 인수(2023년) 이후 체제라 과거 분식과 단절돼 있고, 반복하지 않는다는 신호가 자산이다. KDDX·미국 사업은 신인도가 수주 경쟁력이고, K-SOX는 대표이사가 운영 실태를 보고·책임지는 제도라 경보 체계는 경영진 자신의 방패이기도 하다 |
| 경영진이 고의로 숨기면(입력을 관리하면) 소용없지 않나 | 대우조선의 실패 모드가 고의·묵인이라는 건 인정한다. 그래서 설계 요건이 독립성이다. 경보는 경영진이 아니라 감사위원회·이사회로 직행하고, 입력 누락 자체를 신호로 본다(설계변경 건수와 원가 반영 건수의 괴리 등). 완전 차단 장치가 아니라 묵인 구조가 수년간 유지되기 어렵게 만드는 억제 장치로 주장한다 |
| 한화오션이 직접 만들거나 한화시스템 ICT가 하면 되지 않나 | 공정 예측(물리량)과 충당금 판단(회계)은 층위가 다르다. 계열 SI가 자기 계열을 검증하면 독립성 요건을 채우지 못한다 |
| 스마트야드에 AI가 이미 있다 | 사실이라 인정한다. 제안은 그 위의 회계 환산·경보 레이어다. 인터뷰 결과 현장은 여전히 엑셀·전화·메신저를 병행하고, 경영기획 쪽도 취합·대시보드 자동화와 AI 예측 확대를 인정하면서 숫자의 원인 분석과 경영진 설명은 사람 몫이라고 답했다(인터뷰 2) |
| 신규 함정은 데이터가 없지 않나 | 과거 원가 실적을 학습하는 예측 모델이 아니라 회계기준·감리 사례·계약 문서를 참조하는 도구라 신호 감지와 반영 대조는 동일하게 작동한다. 전례 없는 물량에서는 영향 범위의 폭이 넓어질 뿐이고, 그 폭 자체를 불확실성 표시와 함께 보고한다 |
| 반영 안 한 경보가 쌓이면 오히려 경영진을 겨누는 증거 아닌가 | 성실하게 반영하면 방어 자료, 묵인하면 억제 장치다. 어느 쪽이든 제도 목적에 맞고, 구매를 결정하는 경영진은 성실을 전제로 서명하므로 판매 논리와 모순되지 않는다 |
| 현 감사인 한영이 경쟁사가 설계한 통제를 감사하는 게 문제되지 않나 | K-SOX 통제의 소유·운영자는 회사다. 감사인은 통제의 설계·운영 효과성을 평가할 뿐이고, 외부 전문가가 만든 통제를 회사가 채택하는 것은 일반적 관행이다 |
| 환산 결과가 범위 수준이면 경보 근거로 못 쓰지 않나 | 경보 판정은 정밀 금액이 아니라 보수적으로 잡은 범위의 하한 기준이다. 하한이 임계치를 넘을 때만 경보하므로 범위가 넓어도 판정은 성립한다 |
| 시스템이 과거 왜곡을 찾으면 삼일 감사 실패의 증거가 되니 민감도를 낮출 유인 아닌가 | 민감도·임계치는 삼일이 아니라 감사위원회가 승인하고, 설정값과 변경 이력이 통제 문서로 남는다. 판매자가 조용히 낮출 수 있는 구조가 아니다 |
| 구축·이관이면 결국 SI 아닌가 | 구축은 수단이고 파는 것은 신호 규칙·임계치·검토 게이트라는 판단 체계의 설계와 정기 점검이다. IT 영역 밖이라고 한 것은 구축이 아니라 충당금 판단이다 |
| 경보가 안 울린 기록이 오히려 면책 자료로 악용되지 않나 | 모든 산출물에 커버리지 명세(어떤 문서 범위를 봤는지)를 붙인다. 무경보는 그 범위 안에서의 무경보일 뿐 무혐의 증명이 아니라는 한정을 통제 문서에 명시한다 |
| 구매 주체와 예산은 누구인가 | 내부회계관리제도 운영·개선 예산이고, 승인 주체는 경보 수신자인 감사위원회와 이사회다. 경영진 단독 결정 구조가 아니다 |
| 절벽을 놓치면(미탐) 삼일이 책임지나 | 감사의견이 아니라 통제 도구라 합리적 확신을 파는 것이 아니다. 계약상 역할은 신호 탐지·대조 체계의 설계와 점검이고, 책임 범위도 그 수준으로 한정한다 |
| LLM이 들어간 통제를 감사인이 어떻게 테스트하나 | K-SOX 통제로 등록되는 것은 LLM 자체가 아니라 게이트·경보·승인이라는 절차다. 절차는 표준 통제 테스트로 감사할 수 있고, LLM 산출물은 그 절차의 입력 정보로 취급한다 |
| 대조 기준인 회사 추정치가 월 1회만 갱신되면 상시 대조가 아니지 않나 | 대조 기준은 직전 결산 스냅샷이다. 새 신호가 스냅샷에 없으면 그것이 곧 미반영 신호고, 그 시차가 이 도구의 산출물이다. 스냅샷이 상시 갱신될 필요는 없다 |
| 물량·공법 변경처럼 계약·시장 지표로 계산이 안 되는 신호는 어떻게 하나 | 금액을 지어내지 않는다. 범위를 산출할 수 없다는 사실 자체를 재추정 요구 신호로 올린다 |
| 이관 후 경보 게이트는 누가 맡나 | 운영 단계의 게이트는 사내 회계 조직이 맡고, 삼일은 정기 점검에서 게이트 판단 품질과 임계치·규칙 업데이트(감리 사례 반영)를 리뷰한다 |
| KDDX·미 해군 물량은 군사보안인데 외부 AI가 문서를 읽을 수 있나 | 보안 데이터는 반출하지 않고 사내 구축(온프레미스)으로 처리하며, 신호 규칙과 회계 환산 로직만 외부에서 갱신하는 구조로 답한다. 적용도 상선 물량부터 시작하는 단계론으로 제시 |
| 삼일이 직전 감사에서 적정의견을 냈으면서 이제 와서 위험하다는 건 자기부정 아닌가 | 적정의견은 중요성 관점에서 공정하게 표시됐다는 뜻이지 위험이 없다는 뜻이 아니다. 삼일 감사보고서 스스로 수익인식을 핵심감사사항으로, 러시아 신용위험을 강조사항으로 공개 지적했다. 이 서비스는 그 공개된 위험 지적을 상시 체계로 잇는 연장선이다 |

## 발표에서 하지 말아야 할 주장

한화오션이 부실하다(사상 최대 실적과 모순), 미청구공사가 지금도 늘고 있다(2026년 3월 말 기준 소폭 감소, 고수준 유지로 표현), 선가가 하락세다, 지체상금 소송이 추정 실패의 증거다, 한화오션에 AI 역량이 없다. 전부 반대근거에 반박되는 프레임이라 주장에서 뺐다.

인터뷰의 "예산 줄다리기" 발화를 회계 왜곡의 증언처럼 인용하지 말 것. 어느 조직에나 있는 예산 조율 이야기고, 원가 추정치가 부서 간 협상의 영향을 받는다는 사실까지만 쓸 수 있다.
